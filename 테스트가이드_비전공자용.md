# Eternal Beam 테스트 가이드 (비전공자용)

아래 순서대로 **하나씩** 진행하면 됩니다.  
한 단계가 끝나면 다음 단계로 넘어가세요.

---

## 준비물

- **Windows PC** (Mac이면 일부 명령어만 다르다고 적어 둔 부분이 있습니다)
- **인터넷 연결** (처음 한 번은 프로그램·음악 다운로드에 필요)
- **사진 한 장** (테스트용으로 아무 사진이나, 예: 사람이 있는 사진)

---

## 1단계: 폴더 열기

1. **파일 탐색기**를 연다.
2. 이 프로젝트가 있는 폴더로 간다.  
   예: `C:\Users\choi jungsun\Desktop\eternal-beam-app`
3. 이 폴더 안에 **`input.jpg`** 라는 이름으로 테스트할 **사진 한 장**을 넣는다.  
   - 사진이 다른 이름이면, 그 사진을 **복사**한 다음 이름을 **`input.jpg`** 로 바꿔서 이 폴더에 넣으면 된다.

---

## 2단계: 터미널(명령 프롬프트) 열기

1. 위에서 연 **eternal-beam-app** 폴더 창에서 **주소창(경로가 쓰인 곳)** 을 한 번 클릭한다.
2. 키보드로 **`cmd`** 를 입력한 뒤 **Enter** 를 친다.  
   → 검은 창(명령 프롬프트)이 뜬다.  
   - 이 검은 창이 **“터미널”** 이다.  
   - 여기에 **명령어**를 입력해서 프로그램을 설치하고 실행한다.

---

## 3단계: Python 있는지 확인

1. 검은 창에 아래를 **그대로** 입력하고 Enter를 친다.

   ```
   python --version
   ```

2. **`Python 3.8`** 같은 숫자가 보이면 **Python이 설치된 것**이다.  
   → **4단계**로 간다.

3. **“python은(는) 내부 또는 외부 명령으로 인식되지 않습니다”** 같은 메시지가 나오면:
   - [python.org](https://www.python.org/downloads/) 에서 Python을 설치한 뒤,
   - 설치할 때 **“Add Python to PATH”** 에 체크하고 설치하고,
   - **2단계**부터 다시 한다 (새로 연 cmd 창에서).

---

## 4단계: 가상환경 만들기 (한 번만)

1. 검은 창에서 아래를 **한 줄씩** 입력하고, 각 줄 입력 후 **Enter** 를 친다.

   ```
   python -m venv .venv
   ```

   (끝나면 다음으로)

   ```
   .venv\Scripts\activate
   ```

2. 성공하면 맨 앞에 **`(.venv)`** 같은 것이 붙는다.  
   → 이제 이 창에서 설치하는 것들은 이 프로젝트 전용이다.

---

## 5단계: 필요한 프로그램 설치 (한 번만)

아래 명령어를 **한 줄씩** 입력하고, 각 줄마다 **Enter** 를 친다.  
(복사해서 붙여넣어도 된다.)

```
pip install numpy opencv-python Pillow
```

끝나면 다음:

```
pip install rembg
```

끝나면 다음:

```
pip install torch
```

끝나면 다음 (한 줄만 입력):

```
pip install torchvision
```

끝나면 다음:

```
pip install diffusers transformers accelerate
```

끝나면 다음:

```
pip install PyQt5
```

- **처음**이면 다운로드 때문에 **몇 분** 걸릴 수 있다.  
  에러가 뜨면 **에러 메시지 전체**를 복사해서 개발자에게 보내면 된다.

---

## 6단계: ffmpeg 설치 (한 번만)

- **ffmpeg**는 “영상 만드는 도구”다. Python이 아니라 **컴퓨터에 따로** 깔아야 한다.

### 방법 A – winget 사용 (Windows 10/11)

1. 검은 창에 아래를 입력하고 Enter를 친다.

   ```
   winget install ffmpeg
   ```

2. 설치가 끝나면 **검은 창을 닫았다가**, **2단계**처럼 다시 **eternal-beam-app** 폴더에서 주소창에 `cmd` 입력해서 **새 검은 창**을 연다.
3. 다시 가상환경 켜기:

   ```
   .venv\Scripts\activate
   ```

### 방법 B – winget이 없을 때

1. 브라우저에서 **“ffmpeg download windows”** 로 검색한다.
2. **“gyan.dev”** 또는 **“ffmpeg 공식”** 같은 곳에서 Windows용 ffmpeg를 받는다.
3. 압축을 풀고, **bin** 폴더 경로를 **시스템 환경 변수 PATH**에 추가한다.  
   (방법이 어렵다면 “ffmpeg PATH 추가” 로 검색해서 따라 하면 된다.)

---

## 7단계: 테스트 실행

1. **1단계**를 다시 확인한다:  
   **eternal-beam-app** 폴더 안에 **`input.jpg`** 가 있어야 한다.

2. 검은 창에서 **(.venv) 가 켜진 상태로** 아래를 입력하고 Enter를 친다.

   ```
   python eternal_beam.py
   ```

3. 그러면:
   - 검은 창에 **진행 로그**가 쭉 출력된다.  
     (이미지 로드 → Segmentation → Depth → Inpainting → 프레임 생성 → 인코딩 등)
   - **처음**이면 모델 다운로드 때문에 **5~15분** 정도 걸릴 수 있다.
   - 끝나면 **“저장 완료: output.mp4”** 와 **“전체 화면 재생을 시작합니다”** 가 보인다.

4. 곧 **화면 전체**로 영상이 재생된다.  
   - 6초 정도 재생되고, 잔잔한 피아노 BGM이 나올 수 있다.  
   - **ESC** 키를 누르면 재생이 끝나고 창이 닫힌다.

5. 같은 폴더에 **`output.mp4`** 파일이 생겼다면 **테스트 성공**이다.  
   이 파일을 더블클릭하면 언제든 영상만 다시 볼 수 있다.

---

## 영상 재생이 안 될 때 / 음악(BGM) 넣는 법

### 재생이 안 될 때 (DirectShow 오류 등)

- **“전체 화면 재생을 시작합니다”** 다음에 오류 메시지가 나와도, **output.mp4는 이미 만들어져 있습니다.**
- **eternal-beam-app** 폴더로 가서 **output.mp4** 파일을 **더블클릭**하면 Windows 미디어 플레이어 등 기본 플레이어로 재생됩니다.
- 코드를 수정해서, 내장 재생이 실패하면 자동으로 기본 플레이어가 열리도록 해 두었습니다. (다음 실행부터 적용)

### 음악(BGM)이 안 나올 때

- BGM 다운로드가 403 등으로 실패하면 **무음**으로 저장됩니다.
- **직접 BGM을 쓰고 싶다면**:  
  1. **잔잔한 피아노 곡** mp3 파일을 준비하고,  
  2. 파일 이름을 **`bgm.mp3`** 로 바꾼 뒤,  
  3. **eternal-beam-app** 폴더 **안**에 넣습니다.  
- 다음에 **`python eternal_beam.py`** 를 실행하면 **로컬 bgm.mp3** 를 사용해서 영상에 음악이 들어갑니다. (다시 만들어야 하므로 실행 시간은 그대로 걸립니다.)

## 자주 하는 실수

| 증상 | 확인할 것 |
|------|------------|
| `input.jpg` 를 찾을 수 없습니다 | 1단계: **eternal-beam-app** 폴더 안에 사진을 **`input.jpg`** 이름으로 넣었는지 확인 |
| `python` 을 찾을 수 없습니다 | 3단계: Python 설치 시 **“Add Python to PATH”** 체크 후 재설치 |
| `pip` 을 찾을 수 없습니다 | 4단계: **`.venv\Scripts\activate`** 를 먼저 실행했는지 확인 |
| ffmpeg 관련 오류 | 6단계: ffmpeg 설치 후 **cmd 창을 닫았다가 다시 열고**, 다시 **`.venv\Scripts\activate`** 실행 |
| 화면에 아무것도 안 뜸 | 7단계: **“전체 화면 재생을 시작합니다”** 메시지가 나온 뒤 잠시 기다리기; 그래도 안 되면 PyQt5 재설치 `pip install --upgrade PyQt5` |
