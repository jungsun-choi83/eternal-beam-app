# Eternal Beam — 시스템 아키텍처 감사 보고서

**감사 기준**: Pepper's Ghost 홀로그램 기기 요구사항 vs 현재 코드베이스  
**범위**: 앱(React), 백엔드(FastAPI), Python 재생/파이프라인 스크립트, 문서

## 2. Expected User Experience Flow (상태 흐름)

**요구**: Power ON → Logo → Main subject → Idle → Hand detected → Subject reacts → NFC card → Background changes.

| 단계 | 구현 상태 | 위치 |
|------|-----------|------|
| Power ON | **Partially implemented** | `raspi_crossfade.py`: 부팅 시 Pure Black → 로고 애니메이션 → Idle. 앱에는 해당 플로우 없음. |
| Eternal Beam logo animation | **Implemented correctly** | `raspi_crossfade.py` — Step 2: 페이드 인/유지/페이드 아웃. |
| Main subject appears | **Implemented correctly** | Pi: Idle 영상 재생. 앱: 홈/미리보기에서 cutout 이미지 표시. |
| Idle animation | **Implemented correctly** | `raspi_crossfade.py` — `state == "idle"`, `read_frame_with_loop(idle_cap)`. |
| User hand detected | **Missing** | Pi는 마우스 클릭으로만 트리거. 거리/손 감지 없음. |
| Subject reacts | **Implemented correctly** | Pi: `mouse_triggered` → action 영상 크로스페이드. |
| NFC card inserted | **Missing** | 기기 측에서 NFC 태그 읽어 theme id 추출하는 로직 없음. |
| Background world changes | **Partially implemented** | 앱: 테마 선택으로 배경 변경(UI). Pi: 단일 idle/action만 재생, 슬롯별 배경 전환 미구현. |

**결론**: 앱은 **선형 화면 플로우**(온보딩→업로드→AI처리→테마→미리보기→NFC 저장)만 있고, 기기 전원 켜기부터 NFC로 배경 바꾸기까지의 **통합 상태 머신**은 없음. Pi 쪽은 부팅→로고→Idle↔Action만 구현.

## 11. 라즈베리 파이 배포 전 개선 권장사항

1. **NFC**:  
   - 기기에서 **NFC 리더**로 태그 읽기 (예: py532, nfcpy 또는 하드웨어 SDK).  
   - 슬롯 번호 또는 content_id 저장/조회 API(`get_playback_by_slot`) 구현 후, **슬롯 id → 재생할 배경/피사체 URL** 매핑.

2. **상호작용**:  
   - `raspi_crossfade.py`의 마우스 트리거를 **GPIO/거리 센서** 입력으로 교체.  
   - (선택) approach / react 구간에 맞춘 **idle / approach / react** 영상 3종 재생.

3. **비디오 구조**:  
   - **/videos** (idle, approach, react), **/themes** (sakura, aurora, …) 디렉터리 및 명명 규칙 정하고, Pi 스크립트와 백엔드 URL이 이 구조를 참조하도록 통일.

4. **앱 ↔ 기기 연동**:  
   - **Web NFC** 또는 네이티브 NFC로 **content_id + slot** 실제 기록.  
   - Preview 완료 시 **compose-final** 호출해 content_id 생성 후, 해당 값을 NFC에 씀.

5. **Memorial 앱 파이프라인**:  
   - AI 처리 화면에서 **cutout API** 호출 후, (선택) Runway 연동 또는 백엔드 합성 결과를 **미리보기/재생**에 반영.

6. **Pi 재생기**:  
   - **슬롯/테마 변경** 시 배경·피사체 URL 로드 후, `test_composite_video.py`와 유사한 **2레이어 실시간 합성** 또는 **사전 합성 영상** 재생 로직 추가.

이 문서는 현재 코드베이스 기준 감사 결과이며, 배포 전 위 항목을 보완하면 요구사항에 더 가깝게 맞출 수 있습니다.
