# Eternal Beam / 고야 AI 홀로그램 — 총 기술 사양서 (라즈베리 파이 실험용)

**목적**: Cursor가 구현한 기술 요소를 한 문서로 정리하고, 라즈베리 파이에서 실험할 때 참고하는 사양서.  
**대상**: 개발자·실험자. 요청했던 기능과 구현 상태를 한눈에 파악할 수 있도록 구성.

## 3. 라즈베리 파이에서 실험할 때 (핵심)

### 3.1 재생 스크립트: `raspi_crossfade.py`

- **역할**: **부팅 시퀀스** → 평소 `idle` 영상 무한 루프 → **트리거 시** `action` 영상으로 크로스페이드 → action 종료 후 다시 idle.
- **부팅 시퀀스 (하드웨어 강제)**  
  1. Step 1: 전원 On 후 1초간 **Pure Black(#000000)** 유지  
  2. Step 2: `Eternal Beam` 로고 애니메이션 (중앙 정렬, 배경 Pure Black) → 페이드 인·유지·페이드 아웃  
  3. Step 3: 로고 페이드 아웃 후 즉시 Idle 영상 진입
- **True Black (No Gray Blacks)**: 모든 출력 프레임에 픽셀 클램핑 적용 (흑레벨 0~10 → Pure Black #000000). DLP 프로젝터 투명 처리 최적화.
- **트리거**: 현재 **마우스 클릭**. 이후 **LiDAR/센서**로 교체 예정 (`mouse_triggered = True` 부분만 센서 이벤트로 변경).

**실행 (Pi 또는 PC)**:

```bash
cd python
pip install opencv-python   # Pi: sudo apt-get install python3-opencv 또는 pip
python raspi_crossfade.py
```

**설정 (파일 상단)**:

```python
BASE_DIR = Path(__file__).resolve().parent
IDLE_PATH = str(BASE_DIR / "idle_fix.mp4")   # 또는 idle1.mp4
ACTION_PATH = str(BASE_DIR / "action.mp4")   # 또는 action1.mp4
CROSSFADE_DURATION = 0.8   # 초
TARGET_FPS = 30
```

**소스 준비**:

- **사진을 idle로 쓰려면** 먼저 MP4로 변환:
  ```bash
  ffmpeg -loop 1 -i idle1.png -t 10 -c:v libx264 -pix_fmt yuv420p -movflags +faststart idle1.mp4
  ```
- OpenCV가 못 여는 MP4는 동일하게 재인코딩:
  ```bash
  ffmpeg -i idle.mp4 -c:v libx264 -pix_fmt yuv420p -movflags +faststart idle_fix.mp4
  ```

**찌그러짐 방지 (권장)**  
- 원본 비율 유지 + 검은 패딩이 필요하면 `fit_to_window(frame, w, h)` 함수를 추가하고, `read_frame_with_loop` / action 프레임에 적용. (문서 상단 “비율 유지 + 패딩” 참고.)

### 3.2 라즈베리 파이 홀로그램 재생 시스템 (Supabase / 센서 / 자동 시작)

**목표**: 라즈베리 파이가 부팅되면 자동으로 홀로그램 플레이어가 실행되고,  
NFC에서 읽은 `content_id`와 **모션 센서(GPIO)** 에 반응해서 Idle/Action 영상을 재생하는 구조.

| 항목 | 구현 상태 | 설명 |
|------|----------|------|
| Python 재생 스크립트 | ✅ | `python/raspi_crossfade.py` — 부팅 시퀀스 + Idle/Action 크로스페이드, True Black, 30fps 타이밍 루프 |
| Supabase / 백엔드 API 호출 | 📋 설계 | FastAPI `GET /api/content/{content_id}` 에서 배경/피사체/스케일/포지션 메타데이터 제공. Pi 쪽에서 `requests` 로 호출하여 재생용 MP4(URL) 또는 레이어 URL을 받아 캐싱 → 로컬 파일로 재생 예정 (`python/hardware_playback.py` 등에서 구현). |
| GPIO 센서 연동 (모션 반응) | 📋 설계 | 현재 `raspi_crossfade.py` 는 `mouse_triggered` (마우스 클릭) 를 트리거로 사용. 이 부분을 `RPi.GPIO` 또는 `gpiozero` 로 읽은 센서 입력(HIGH/LOW)으로 치환하면, 센서 감지 시 Action, 미감지 시 Idle 로 동작. |
| 부팅 시 자동 시작 | 📋 설계 | `/etc/systemd/system/eternal-beam.service` 를 만들어 부팅 시 `python3 /home/pi/eternal-beam-app/python/raspi_crossfade.py` (또는 `hardware_playback.py`) 가 자동 실행되도록 설정. `systemctl enable/start eternal-beam.service`. |
| 30fps 이상 재생 | ✅ (라즈베리 파이 4 기준) | `TARGET_FPS = 30` 으로 메인 루프 타이밍을 제어. Idle/Action 영상을 H.264, 720p/1080p, 30fps 로 인코딩하면 Pi 4에서 30fps 유지 가능. |

**예시: systemd 서비스 (참고용)**  
라즈베리 파이 OS에서 부팅 자동 실행 설정 예:

```ini
[Unit]
Description=Eternal Beam Player
After=network.target

[Service]
ExecStart=/usr/bin/python3 /home/pi/eternal-beam-app/python/raspi_crossfade.py
WorkingDirectory=/home/pi/eternal-beam-app
Restart=always
User=pi

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable eternal-beam.service
sudo systemctl start eternal-beam.service
```

이 섹션은 **GitHub에서 다른 개발자들이 Pi 재생 시스템을 이해하고 이어받을 때 참고용 사양**이다.  
추후 `hardware_playback.py` 에 Supabase API 호출 + GPIO 센서 입력을 묶어 구현하면, 이 표의 📋 항목을 ✅ 로 변경하면 된다.

## 4. 앱 (React) — 실행·플로우

- **실행**: `npm run dev` → 터미널에 나온 주소 (예: `http://localhost:5174/`) 를 브라우저 주소창에 입력.
- **플로우**: 온보딩 → (선택) 로그인 → 홈 → 사진 업로드 → AI 처리(누끼 API) → 테마 선택 → 미리보기 → 결제(프리미엄 시) → NFC 저장/재생.
- **백엔드 연동**: `.env`에 `VITE_VIDEO_API_URL` 또는 `VITE_API_URL` (예: `http://localhost:8000`) 설정 시 누끼·합성 API 사용.

---

## 5. 백엔드 (FastAPI) — API·역할

- **실행**: `cd backend && pip install -r requirements.txt && uvicorn main:app --reload` (기본 포트 8000).
- **주요 API**:
  - `POST /api/cutout`: 이미지 업로드 → rembg(u2net_human_seg) + Alpha Matting → cutout PNG URL 또는 base64.
  - `POST /api/compose-video`: cutout 파일 + theme_id + payment_status → FFmpeg 합성(Soft Glow, Breathing) → unique_url, nfc_payload.
- **의존성**: rembg, Pillow, opencv-python-headless, ffmpeg(시스템), (선택) Supabase.

## 6. PC용 MVP 파이프라인 (`eternal_beam_mvp.py`)

- **역할**: `input.jpg` → 피사체 분리(U2-Net+Alpha Matting) + (선택) ZoeDepth → FFmpeg(테마+Breathing+Soft Glow+DoF+True Black+Film Grain+BGM) → `output.mp4` + 완료 후 전체 화면 재생.
- **실행**: `cd python && pip install -r requirements-mvp.txt` 후 `python eternal_beam_mvp.py input.jpg [output.mp4]`.
- **옵션**: `--theme`, `--duration`, `--bgm`, `--full-hd`, `--no-popup`. 자세한 내용은 `python/README-MVP.md` 참고.

## 7. 고야 AI 홀로그램 3단계 설계 (참고)

1. **생명력 부여**: DeeVid AI 등으로 사진 → 1080P 주행 영상(고야.mp4).
2. **레이어 최적화**: 고야.mp4에서 주인공만 선명, 배경은 어둡게 → 홀로그램 전용 영상. (Cursor/FFmpeg 보정.)
3. **공간 구현**: NFC 슬롯 + 빔 프로젝터. 슬롯 인식 시 해당 테마 영상 재생.  
   **센서**: 거리 감지 — 기기 폭 18cm 기준으로 재설계.  
   - **15cm 이내**: 손 감지 → idle → action 전환 (기존 30cm에서 변경)  
   - **10cm 이내**: action 유지 또는 강도 강화 (얼굴 들이댐 등)  
   - `raspi_nfc_playback.py`의 `DISTANCE_THRESHOLD_CM = 15.0`으로 적용됨. Pi 현장 실측 후 10~20cm 사이로 조정 권장.  
   → **현재 구현**: `raspi_crossfade.py`의 트리거만 마우스 → `raspi_nfc_playback.py`에서 GPIO/센서 스텁 구현. Pi 연결 후 `is_hand_near()` 함수 내부 교체하면 동일 로직 적용 가능.  
   → **NFC 슬롯**: 상단 삽입 방식으로 변경. 소프트웨어 동작에는 영향 없음 (카드 UID 감지 방식 동일).

## 8. 영상·오디오 포맷 (기기 재생)

- **영상**: H.264, 720p/1080p, yuv420p, MP4. (상세는 `docs/하드웨어_요구_사양서.md` 4장.)
- **오디오**: AAC, 44.1kHz, 스테레오.
- **True Black (하드웨어 강제)**  
  - Pixel Clamping: 흑레벨 0~10 범위를 Pure Black(#000000)으로 강제 조정 (FFmpeg `curves=all='0/0 0.04/0 1/1'` 또는 OpenCV 동등 처리).  
  - DLP 프로젝터 최적화: `-color_range pc` 등으로 검은색을 빛 없음으로 인코딩 → 프로젝터 투명 처리.  
  - 적용 위치: `raspi_crossfade.py`, `ffmpegSynthesizer.js`, `compose_video_service.py`, `eternal_beam_mvp.py`.

## 9. NFC·슬롯 (앱 ↔ 기기)

- **앱**: NFC로 payload 기록 (video_id, content_id, slot, timestamp 등). `nfcManager`, NFCPlayback 화면.
- **기기(Pi)**: 슬롯 번호 읽기 → Supabase `get_playback_by_slot(slot)` 또는 Edge Function으로 재생 URL 조회 → 영상 재생. (현재 Pi 스크립트는 로컬 파일 재생만 구현.)
