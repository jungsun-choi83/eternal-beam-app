### 4) NFC (4.15 전에 주문·도착시키기)

| 용도 | 추천 | 비고 |
|------|------|------|
| **PC/앱에서 태그 쓰기** | 스마트폰 NFC (Web NFC 지원 브라우저) 또는 USB NFC 리더 (ACR122U 등) | 4.15 전에 1대 준비 |
| **라즈베리 파이에서 태그 읽기** | PN532 (SPI/I2C) 또는 RC522 | Pi에 연결해 “카드 꽂으면 theme id 읽기” 테스트용 |

**지금 할 일**: Pi용 NFC 리더 + NFC 카드(빈 태그) 2~4장 주문해 4.15 전 도착시키기.

### 5) 거리 센서 (Pi 상호작용용, 4.15 전 도착 목표)

| 종류 | 예시 | 용도 |
|------|------|------|
| 초음파 (HC-SR04) | 저렴 | 손 가까이 대면 반응 |
| ToF (VL53L0X 등) | 더 정확 | 같은 용도 |

**지금 할 일**: Pi에 연결할 센서 1개 주문.

### 6) 라즈베리 파이 (이미 있으면 스킵)

- Raspberry Pi 4 (2GB 이상 권장)
- microSD 카드, 전원, HDMI
- (선택) 터치스크린 또는 모니터

### 7) 환경 변수·설정 파일

- `.env`에 `RUNWAY_API_KEY`, `VITE_VIDEO_API_URL`(백엔드 주소) 확인.
- 백엔드 실행 시 Supabase URL/Key 있으면 설정.

### Phase 3 — Pi에서 NFC 읽기 + 슬롯 → 영상 매핑 (1~2일)

| # | 작업 | 위치 | 완료 시 |
|---|------|------|--------|
| 3.1 | **get_playback_by_slot** API 또는 로컬 JSON/테이블: slot_number → background_url, subject_url (또는 합성 영상 URL) | 백엔드 `routers/content.py` 또는 새 엔드포인트 | 슬롯 번호로 재생할 URL 조회 가능 |
| 3.2 | Pi 스크립트에서 **NFC 리더**로 태그 읽기 → content_id 또는 slot_number 추출 | 새 파일 예: `python/raspi_nfc_playback.py` | 카드 꽂으면 slot/content_id 출력 |
| 3.3 | 읽은 slot/content_id로 **배경·피사체 URL** 로드 후 재생 (기존 `test_composite_video.py` 로직 활용 또는 단일 합성 영상 재생) | 같은 Pi 스크립트 | NFC 카드 바꾸면 배경/피사체 전환 |

**테스트**: Pi + NFC 리더 + 카드로 슬롯 읽기 → 해당 테마/피사체 재생되는지 확인.
