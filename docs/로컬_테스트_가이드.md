# AI 합성 로컬 테스트 가이드

하드웨어 없이 AI 합성(3D 변환) 흐름을 테스트하는 방법입니다.

---

## 1. 사전 준비

### FFmpeg 설치

백엔드 합성에 FFmpeg가 필요합니다.

- **Windows**: https://ffmpeg.org/download.html 에서 다운로드 후 `ffmpeg`를 PATH에 추가
- 또는: `winget install ffmpeg` (Windows 10/11)
- 설치 확인: 터미널에서 `ffmpeg -version` 입력

---

## 2. 로컬 테스트 실행

### 1) 백엔드 서버 실행

```bash
cd c:\Users\choi jungsun\Desktop\eternal-beam-app
npm run server
```

`http://localhost:3001` 에서 서버가 뜹니다.

### 2) 프론트엔드 실행 (새 터미널)

```bash
cd c:\Users\choi jungsun\Desktop\eternal-beam-app
npm run dev
```

`http://localhost:5174` 로 앱이 열립니다.

### 3) 앱에서 테스트 흐름

1. **사진 업로드** → 사진 선택 (또는 슬롯 업로드 경로)
2. **테마 선택** → 원하는 테마 선택
3. **음성 녹음** → 녹음 버튼으로 음성 녹음
4. **합성 후 다음** → 합성 API 호출 (백엔드 동작)
5. **미리보기** → 합성된 영상 확인

---

## 3. 확인 포인트

| 단계 | 확인 |
|------|------|
| 사진/테마 선택 | 화면 전환이 정상인지 |
| 음성 녹음 | 녹음 시작/중지 동작 |
| 합성 버튼 | 클릭 시 진행률 표시 |
| 미리보기 | 합성 영상 재생 가능 여부 |

---

## 4. 백엔드 배포 후 테스트

로컬 테스트가 잘 되면, Railway / Render 등에 백엔드를 배포한 뒤:

1. `.env`에 `VITE_API_BASE=https://your-api-url` 추가
2. `synthesisService.ts`의 `API_BASE`를 해당 URL로 변경
3. Vercel에 재배포
4. 배포된 앱에서 동일 흐름 테스트
