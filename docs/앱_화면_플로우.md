# Eternal Beam 앱 화면 플로우 (재설계 반영)

비즈니스 핵심 가치 **'아이(피사체) + 슬롯(테마) 판매'** 중심으로 단일 선형 플로우로 재설계된 구조입니다.  
**22개 화면 → 8개 핵심 플로우**로 통합되었습니다.

---

## 1. 8개 핵심 플로우

| # | 플로우 | 화면 ID | 설명 |
|---|--------|---------|------|
| 1 | 온보딩/인증 | onboarding, signup, login | 최초 진입 및 로그인 |
| 2 | 기기 연동 | qrConnection, deviceConnection | QR/Wi-Fi 기기 등록 |
| 3 | **홈** | **home** | 아이 중앙 + 테마 슬롯 가로 스크롤, 실시간 Canvas 프리뷰 |
| 4 | 사진 → AI | photoUpload, aiProcessing | 사진 업로드 → Step 1 누끼 → Step 2 Runway 모션 → Step 3 최적화 → 테마 선택 |
| 5 | 테마 선택 | themeSelection | 슬롯(테마) 선택, 프리미엄 시 결제 Gate |
| 6 | **미리보기** | **preview** | **실시간 조정(Scale/Position)** + 프리뷰(디바운스 800ms) + 완료 시 **Content_ID 생성** → NFC |
| 7 | 결제 | checkout | 결제 완료 시 setThemePaid → nfcPlayback |
| 8 | NFC 저장/재생 | nfcPlayback | **Content_ID만** NFC에 기록 → 슬롯 매핑 |

## 3. 단일 선형 플로우

```
photoUpload → aiProcessing → themeSelection → preview → [checkout(프리미엄 시)] → nfcPlayback
```

- **photoUpload**: 사진/동영상 선택 후 → 항상 **aiProcessing**으로.
- **aiProcessing**: 서버 **누끼 API** 호출 → **Step 1 누끼** 완료 → **Step 2 Runway 모션** (모션 생성) → **Step 3 최적화** → "아이와 재회를 준비 중입니다..." 문구, 완료 시 → **themeSelection**.
- **themeSelection**:  
  - 무료/보유 테마 선택 후 "다음: 미리보기" → **preview**.  
  - 프리미엄 미보유 테마 선택 시 → **checkout** 리다이렉트.
- **preview**: **실시간 조정 UI** — Scale(0.5~2.0), Position X/Y(-100~100), 리셋 버튼. 프리뷰는 800ms 디바운스로 갱신. "완료" 클릭 시 **Content_ID 생성**(composeFinal API) 후 → **nfcPlayback**.
- **checkout**: 결제 성공 시 `payment_status` 반영 및 `setThemePaid(themeId)` 후 → **nfcPlayback**.
- **nfcPlayback**: **Content_ID만** NFC 태그에 기록(compose-video API 호출 없음). 슬롯 선택 후 전송 → 재생 안내.

## 4. 백엔드 영상 처리 (SSR) 및 Content_ID

- **누끼**: `POST /api/cutout` — rembg + Alpha Matting(머리카락 한 올까지) → PNG 또는 Storage URL.
- **프리뷰**: `POST /api/preview` — Scale/Position 적용 실시간 프리뷰(5초).
- **Content_ID**: `POST /api/compose-final` — 메타데이터 저장 후 **content_id** 반환. NFC에는 **Content_ID만** 기록.
- **합성**(선택): `POST /api/compose-video` — 테마 MP4 + 누끼 오버레이(기존 플로우에서 사용 시).
- **NFC**: **Content_ID**(+ slot_number)만 JSON으로 NFC 태그에 기록. 기기는 Content_ID로 레이어 URL 조회(`GET /api/content/{content_id}`).

### 4-1. 아키텍처 방향 (실서비스)

- **고성능 백엔드 서버(FastAPI)** 가 rembg/FFmpeg를 사용해 **누끼 + 합성**을 처리하고,  
  라즈베리 파이/기기에는 **완성된 소스(Idle/Action 영상)만 스트리밍/다운로드** 하는 구조가 목표.
- 라즈베리 파이는 **재생 전용**으로 동작하며, 무거운 AI/FFmpeg 처리는 모두 서버 측에서 수행.

### 4-2. 데모 모드 (노트북 로컬 테스트용)

- 로컬 환경에서 백엔드 누끼 서버 연결이 불안정하거나 느린 경우를 위해,  
  `AIProcessingScreen.tsx` 에서 `cutoutImage()` 호출이 실패해도:
  - `catch` 블록에서 **원본 이미지 URL을 그대로 `eternal_beam_cutout_url`** 로 저장.
  - 누끼 이미지는 아니지만, 이후 **테마 선택 → 미리보기 → Content_ID → NFC 저장** 플로우는 동일하게 테스트 가능.
- 즉, **실제 서비스**에서는 서버 누끼 결과를 사용하고,  
  **데모 모드**에서는 원본 이미지를 피사체처럼 취급해 전체 UX 흐름을 검증한다.

## 5. 홈 화면 (실시간 프리뷰)

- **중앙**: 현재 선택된 '아이' 실루엣/합성 영상 + 글로우.
- **[사진 교체]**: 클릭 시 **photoUpload**로 이동.
- **하단**: 테마 슬롯 가로 스크롤 카드.
  - 무료/보유 테마 클릭 → **Canvas/CSS 기반 실시간 프리뷰**(서버 연산 없이 배경 변경).
  - 유료 미보유 테마 클릭 → 자물쇠 + 결제 팝업 → **checkout**.
- **NFC 슬롯에 저장하기**: 피사체 있으면 **preview**로, 없으면 **photoUpload**로.
