# Eternal Beam — 추가로 고칠 사항 (백로그)

수동으로 API 키 넣어 Unity에서 테스트하는 동안, 나중에 손볼 이슈만 여기 적어 둡니다.

## 누끼 품질 (전처리)

- **현상:** YOLO로 강아지 영역만 잘라 rembg(방식 A)하면 털 경계가 얼룩덜룩하게 나오는 경우가 있음. 전체 프레임 rembg(방식 B)는 더 깔끔함.
- **방향:** 기본 파이프라인을 **전체 rembg 우선**으로 할지, `use_yolo_crop` 같은 플래그로 A/B 선택할지 결정.
- **참고:** 손·사람 제외가 필요하면 “작은 크롭만 rembg”보다 **전체 rembg + 가장자리 후처리** 또는 **넓게 패딩한 크롭 rembg**가 현실적일 수 있음.

## Luma API

- **현상:** 계정에 크레딧이 없으면 API 키 생성 자체가 막힐 수 있음(“Purchase credits to create an API key”).
- **방향:** 운영/스테이징별로 크레딧·키 정책 정리.

## 기타 (필요 시 아래에 추가)

- Windows에서 AnySign4PC가 `.env`만 암호화하는 경우 → `backend/env.local` 사용 (프로젝트가 로드함).
