# 저장소 및 플랜 설정 가이드

## 1. Supabase 마이그레이션 실행

1. Supabase **SQL Editor**에서 `supabase/migrations/20250220000000_user_quotas.sql` 내용 전체 복사
2. 붙여넣기 후 **Run** 실행

## 2. Supabase Storage 버킷 생성

1. Supabase 대시보드 → **Storage** → **New bucket**
2. 이름: `user-media`
3. **Public bucket** 체크 해제 (비공개)
4. 생성

## 3. 서버 환경 변수 (.env 또는 server 실행 시)

```env
SUPABASE_URL=https://프로젝트ID.supabase.co
SUPABASE_SERVICE_ROLE_KEY=서비스_롤_키
```

- **Service Role Key**: Supabase **Project Settings** → **API** → `service_role` (secret)

## 4. 플랜 제한

| 플랜 | 월 가격 | 생성 횟수 | 저장 용량 | 해상도 |
|------|---------|-----------|-----------|--------|
| 베이직 | ₩14,900 | 10회 | 5GB | 720p |
| 프리미엄 | ₩29,900 | 30회 | 20GB | 1080p |
| 평생 소장권 | ₩490,000 (일시불) | 30회 | 50GB | 1080p |

## 5. 월 리셋 스케줄러

매월 1일 00:00에 `generation_count_this_month` 리셋:

```bash
supabase functions deploy reset_monthly_quotas
```

- **Supabase Cron** (Pro 플랜): Database → Extensions → pg_cron
- **외부 Cron**: 매월 1일 `POST https://프로젝트.supabase.co/functions/v1/reset_monthly_quotas` 호출
