# 설계자 PDF 기반 – 앱 기술 체크리스트

설계 문서(반려동물/아이돌 추모 홀로그램 디스플레이)와 현재 앱을 비교한 **추가·정리 필요 기술 항목**입니다.

---

## 1. 앱 콘텐츠 생성 (App Experience)

### 1.1 사진 업로드
| 설계 요구 | 현재 상태 | 비고 |
|-----------|-----------|------|
| 갤러리에서 반려동물/IP 캐릭터 사진 선택 | ✅ 구현됨 (PhotoUploadScreen) | - |

### 1.2 AI 영상 변환 (Key 기술)
| 설계 요구 | 현재 상태 | 추가 필요 |
|-----------|-----------|-----------|
| 피사체와 배경 **분리** (Segmentation) | ⚠️ 시뮬레이션만 (타이머 + 가짜 video_id) | **실제 AI API 연동** (백엔드 또는 외부 서비스) |
| **Depth Map** 생성으로 입체감 | ⚠️ UI 문구만 ("깊이 맵 생성 중...") | Depth Map 생성 API/파이프라인 |
| **인페인팅(In-painting)**으로 숨겨진 뒷면 유추 | ❌ 미구현 | 3D/유사 3D 영상 생성 백엔드 |
| 최종 **유사 3D 영상** 산출물 | ⚠️ `holo_xxx` ID만 저장, 실제 영상 URL 없음 | 영상 파일 생성 + 스토리지 URL 저장 |

**권장:**  
- 백엔드에서 “사진 → 세그멘테이션 → Depth Map → 인페인팅 → 영상 인코딩” 파이프라인 구현 후, 앱은 **실제 영상 URL**을 받아 `eternal_beam_hologram_video_id` 및 스토리지 URL로 저장하도록 연동.

## 2. 물리적 연결 (Slot & Hardware) – 앱 관점

### 2.1 슬롯 주문/인식
| 설계 요구 | 현재 상태 | 추가 필요 |
|-----------|-----------|-----------|
| 사용자가 선택한 **테마의 NFC 슬롯**을 기기에 꽂음 | ✅ 앱에서 슬롯 번호 선택 후 NFC **쓰기** | - |
| **기기 → 앱** 신호: 기기가 슬롯 ID를 읽어 "02번(크리스마스) 테마 실행" 전달 | ❌ 미구현 | **기기-앱/서버 통신** |

**설계서 의미:**  
- 기기(라즈베리 파이 + NFC 리더)가 슬롯을 꽂으면 **슬롯 ID(예: 02)**를 읽고,  
  앱 또는 서버에 “슬롯 02 재생 요청”을 보내는 흐름이 필요합니다.

**권장:**  
- 기기용 클라이언트(라즈베리 파이)에서 **REST/WebSocket** 등으로  
  `POST /api/device/slot-detected` 같은 API에 `{ "slot_number": 2 }` 전송.  
- 앱/백엔드는 `slot_number` ↔ `content_id` 또는 `video_id` 매핑 테이블로 해당 콘텐츠 URL 반환.

### 2.2 데이터 전송
| 설계 요구 | 현재 상태 | 추가 필요 |
|-----------|-----------|-----------|
| 앱/클라우드에서 생성된 **3D 영상**을 **Wi-Fi**로 기기(메인보드)에 전송 | ❌ 미구현 | **기기용 수신 API/클라이언트** |

**설계서 의미:**  
- “앱은 클라우드에서 생성된 3D 영상을 Wi-Fi를 통해 기기로 전송”  
- 즉, **기기(Raspberry Pi)**가 Wi-Fi로 영상(스트리밍 또는 다운로드)을 받아 재생해야 함.

**권장:**  
- **앱/백엔드:**  
  - 콘텐츠별 **영상 URL**(및 오디오 URL)을 Firebase/스토리지에 저장하고,  
  - 기기 요청 시 해당 URL을 내려주는 **API** 제공.  
- **기기 측:**  
  - 슬롯 ID 수신 → API 호출 → 영상 URL 수신 → 다운로드/스트리밍 재생하는 **별도 프로그램**(Python 등) 필요.  
  - 이 부분은 “앱” 코드가 아니라 **기기 펌웨어/스크립트** 영역입니다.

## 3. 테마 ↔ 슬롯 매핑

| 설계 요구 | 현재 상태 | 추가 필요 |
|-----------|-----------|-----------|
| 슬롯 번호(예: 02) ↔ 테마(예: 크리스마스) 일치 | ⚠️ 사용자가 슬롯 번호만 선택, “테마 이름”은 메타데이터로만 저장 | **슬롯 번호 ↔ 테마 ID/이름** 공식 매핑 (기기·앱·백엔드 공통) |

**권장:**  
- `content_permissions` 또는 `contents`에 `slot_number`, `background_theme_id` 저장해 두고,  
  기기가 “슬롯 2”라고 보내면 서버가 “슬롯 2에 해당하는 content_id / video_url”을 반환하도록 설계.

## 4. 영상·오디오 포맷 (DLP/기기 호환)

| 설계 요구 | 현재 상태 | 추가 필요 |
|-----------|-----------|-----------|
| DLP 광학 모듈로 투사 가능한 영상 포맷 | ❌ 앱에서 포맷/해상도 정의 없음 | **기기 재생 스펙 정의** (해상도, 코덱, 프레임, 파일 형식) |
| 음성·효과음 출력 (스피커) | ✅ 앱에서 음성+BGM 믹싱 후 `mixed_audio` 저장 | 기기 전송 시 **오디오 URL 포함** 여부만 확인 |

**권장:**  
- 백엔드/AI 파이프라인에서 DLP 또는 재생기 스펙에 맞는 **영상 인코딩**(해상도, 코덱 등) 규격을 정하고,  
  앱은 “이 콘텐츠의 재생 가능 영상 URL”만 받아서 저장·전달하도록 하면 됩니다.

## 5. 요약 – 앱에 추가되어야 할 기술 요소

1. **AI 영상 변환 실체화**  
   - 배경 분리, Depth Map, 인페인팅을 수행하는 **백엔드/API** 연동.  
   - 앱은 최종 **영상 URL**을 받아 저장하고, 이후 단계에서 사용.

2. **기기 → 앱/서버 통신**  
   - 기기가 NFC로 읽은 **슬롯 ID**를 서버에 전달하는 **API** 설계.  
   - 서버는 슬롯 ID에 맞는 **영상 URL(및 오디오 URL)**을 기기에 반환.

3. **앱/클라우드 → 기기 전송**  
   - 기기가 Wi-Fi로 영상을 받을 수 있도록 **URL 기반 다운로드/스트리밍** 지원.  
   - 실제 전송·재생은 **기기 측 클라이언트**에서 구현.

4. **슬롯 ↔ 테마 공식 매핑**  
   - DB/설정에 “슬롯 번호 ↔ content_id / theme_id” 매핑을 두고,  
   앱·기기·백엔드가 동일 규칙을 사용하도록 정리.

5. **영상/오디오 포맷 스펙**  
   - DLP 및 재생 환경에 맞는 **해상도·코덱·포맷** 문서화 후,  
   AI 파이프라인과 기기 재생 로직이 이 스펙을 따르도록 함.

---

## 6. 현재 앱에서 잘 맞는 부분

- 사진 업로드 → (시뮬레이션) AI 처리 → 테마 선택 → 합성 미리보기 → 음성 녹음·믹싱 → 결제 → **NFC 카드 쓰기**(video_id, content_id, slot) → 재생 완료 화면  
- 설계서의 **“사용자가 테마를 선택하고, 그에 맞는 슬롯을 꽂는다”**는 UX와 방향이 같음.  
- 부족한 부분은 **실제 AI 영상 생성**, **기기가 슬롯을 읽어 서버에 알리는 흐름**, **기기로의 영상 전달**입니다.
