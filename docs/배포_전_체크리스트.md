| 항목 | 상태 | 메모 |
|------|------|------|

## 4단계: 백엔드 서버 배포 (FFmpeg 합성 API)

현재 앱은 **프론트엔드 + 백엔드(합성 API)** 두 부분으로 구성됩니다.

- **프론트엔드**: Vercel / Netlify (정적 빌드)
- **백엔드**: **FFmpeg가 설치된 서버** 필요 (Railway, Render, Fly.io, VPS 등)

| 항목 | 상태 | 메모 |
|------|------|------|
| 백엔드 호스팅 플랫폼 선정 | | Railway, Render, Fly.io, EC2 등 |
| FFmpeg 설치 가능 환경 선택 | | 대부분의 PaaS는 FFmpeg 포함 |
| 환경 변수: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` | | 서버 배포 시 설정 |
| `SERVER_PORT` (선택, 기본 3001) | | |
| 배포 후 API URL 확인 | | 예: `https://your-api.railway.app` |

---

## 5단계: 클라이언트 API URL 연동

배포된 백엔드 URL을 앱에 설정해야 합니다.

| 항목 | 상태 | 메모 |
|------|------|------|
| `synthesisService.ts`의 API_BASE | | 개발: `''` (프록시), 배포: `https://your-api-url` |
| Vite proxy → 프로덕션에서는 사용 불가 | | 반드시 실제 API URL로 변경 |

---

## 6단계: 합성 결과 URL 처리

Supabase Storage 사용 시 `outputUrl`이 객체 경로입니다. 재생하려면:

| 항목 | 상태 | 메모 |
|------|------|------|
| Signed URL 또는 Public URL 반환 | | 서버에서 `createSignedUrl` 사용 중 |
| 클라이언트가 `signedUrl` 우선 사용 | | `localStorage`에 저장 시 URL이 재생 가능한지 확인 |

---

## 7단계: 동영상 입력 합성 (선택)

현재 RecordScreen → synthesize는 `imageBlob`만 전달합니다. 동영상 업로드 시:

| 항목 | 상태 | 메모 |
|------|------|------|
| `synthesisService`에 video Blob 전달 | | FormData에 `video` 필드 추가 |
| 서버 `ffmpegSynthesizer` | | 이미 `videoPath` 지원됨 ✅ |
