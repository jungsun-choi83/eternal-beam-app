# Eternal Beam 하드웨어 요구 사양서 (Hardware Requirements Specification)

**문서 버전**: 1.0  
**작성일**: 2025-02  
**대상**: 프로젝터·홀로그램 디스플레이 하드웨어 제작 업체

---

## 1. 개요

Eternal Beam은 **사진/동영상 + 음성 메시지**를 합성하여 **3D 홀로그램 콘텐츠**를 만들고, 사용자가 **프로젝터** 또는 **NFC 슬롯**에 전송·재생하는 서비스입니다.

본 문서는 하드웨어 업체가 프로젝터/디스플레이 제작 시 **필수로 알아야 할 기술 사양**을 정리한 것입니다.

---

## 2. 전체 시스템 구조

```
[사용자 스마트폰 앱]
        │
        ├─▶ 사진/동영상 업로드
        ├─▶ 음성 녹음 + 배경음악 믹싱
        ├─▶ 영상 합성 (서버 FFmpeg)
        ├─▶ Supabase Storage에 MP4 저장
        ├─▶ 결제 (토스페이)
        │
        ▼
[저장 위치 선택]
        │
        ├── 하드웨어 저장 ──▶ QR 코드로 기기 연결 ──▶ 프로젝터로 영상 전송
        │
        └── 슬롯 저장 ──────▶ 결제 후 NFC 슬롯에 콘텐츠 ID 기록
                                    │
                                    ▼
                             [프로젝터/디스플레이]
                             NFC 슬롯 인식 → 슬롯 번호로 재생 URL 조회 → 영상 재생
```

---

## 3. 하드웨어 유형별 요구 사양

### 3.1 프로젝터형 (QR 연결)

- **역할**: 사용자가 앱에서 QR 코드를 스캔하여 기기와 연결한 뒤, 합성된 영상을 Wi-Fi 등으로 전송·재생
- **필수 기능**:
  - QR 코드 생성·표시 (기기 ID 포함)
  - Wi-Fi 통신 (영상 파일 수신)
  - **영상 재생** (아래 4장 포맷 준수)

### 3.2 NFC 슬롯형 (결제 후 사용)

- **역할**: 사용자가 결제 후 NFC 태그에 콘텐츠 ID를 기록. 프로젝터가 NFC 슬롯을 읽으면 해당 콘텐츠 영상을 재생
- **필수 기능**:
  - **NFC 리더** (슬롯 번호 인식)
  - 인터넷 연결 (Supabase에서 재생 URL 조회)
  - **영상 스트리밍·재생** (아래 4장 포맷 준수)

---

## 4. 영상/오디오 사양 (필수)

### 4.1 영상 포맷

| 항목 | 사양 | 비고 |
|------|------|------|
| **코덱** | H.264 (AVC) | baseline profile, level 3.0 |
| **해상도** | 최대 720p (베이직) / 1080p (프리미엄·평생) | Pi Zero 2W 호환 |
| **프레임** | 가변 | - |
| **비트레이트** | 영상 1~1.5 Mbps | - |
| **컨테이너** | MP4 | - |
| **픽셀 포맷** | yuv420p | - |

### 4.2 오디오 포맷

| 항목 | 사양 |
|------|------|
| **코덱** | AAC |
| **샘플레이트** | 44100 Hz |
| **채널** | 2 (스테레오) |
| **비트레이트** | 128 kbps |

### 4.3 파일 특성

- **입력 소스**: 정적 이미지(JPG, PNG) 또는 동영상(MP4, WebM)
- **출력**: 이미지 + 음성/배경음악 합성 MP4 (최대 길이: 음성 길이 기준, 페이드아웃 2초)
- **예상 파일 크기**: 10~50 MB (길이·해상도에 따라 상이)

---

## 5. NFC 인터페이스

### 5.1 NFC 쓰기 (스마트폰 → NFC 태그)

- **표준**: Web NFC API (NDEF)
- **기록 형식**: Text Record (JSON)
- **JSON 구조**:
```json
{
  "video_id": "holo_xxx",
  "content_id": "content_xxx",
  "slot": 1,
  "timestamp": "2025-02-20T12:00:00.000Z"
}
```

- **슬롯 번호**: 1, 2, 3, … (정수, 슬롯당 1개 콘텐츠 매핑)

### 5.2 NFC 읽기 (프로젝터 → 슬롯 인식)

- 프로젝터는 **슬롯 번호**를 인식해야 함
- 슬롯 번호로 **Supabase API**를 호출하여 재생할 `video_url`을 조회
- 조회 API: `get_playback_by_slot(p_slot_number)` (아래 6장)

---

## 6. 클라우드 연동 (Supabase)

### 6.1 프로젝터가 호출하는 API

**RPC 함수**: `get_playback_by_slot(p_slot_number INTEGER)`

- **입력**: 슬롯 번호 (1, 2, 3, …)
- **출력** (한 행):
  - `content_id`: 콘텐츠 식별자
  - `video_url`: 재생할 MP4 URL (Signed URL 또는 Public URL)
  - `mixed_audio_url`: 믹싱된 오디오 URL (선택)
  - `composed_preview_url`: 합성 미리보기 URL (선택)
  - `background_theme_name`: 테마명 (선택)

- **호출 예 (Supabase 클라이언트)**:
  ```javascript
  const { data } = await supabase.rpc('get_playback_by_slot', { p_slot_number: 2 })
  const videoUrl = data[0].video_url
  ```

- **Edge Function 대안** (HTTP):
  ```
  GET https://{PROJECT_REF}.supabase.co/functions/v1/get_playback_by_slot?slot=2
  Authorization: Bearer {ANON_KEY}
  ```

### 6.2 데이터 저장 구조 (참고)

- **contents**: 콘텐츠별 영상 URL, 오디오 URL 등
- **slot_content_mapping**: 슬롯 번호 ↔ content_id 매핑
- **user_media / user_quotas**: 사용자 저장 용량·생성 횟수 제한 (플랜별)

## 7. 네트워크 요구 사항

| 항목 | 요구 사항 |
|------|-----------|
| **인터넷** | Wi-Fi 또는 유선 (Supabase API 호출, 영상 스트리밍) |
| **방화벽** | HTTPS(443) 허용 (Supabase: `*.supabase.co`) |
| **대역폭** | 영상 스트리밍용 (1~1.5 Mbps × 동시 재생 수) |

---

## 8. 플랫폼별 최적화

### 8.1 라즈베리 파이 제로 2W (참고)

- H.264 baseline, level 3.0
- 720p 이하 권장 (베이직 플랜)
- 1080p 지원 시 HW 디코딩 사용 권장

### 8.2 기타 임베디드

- H.264 디코딩 하드웨어 가속 권장
- AAC 디코딩 지원 필수

---

## 9. 사용자 플로우 요약

| 단계 | 사용자 행동 | 하드웨어 관점 |
|------|-------------|---------------|
| 1 | 사진/동영상 업로드, 음성 녹음 | - |
| 2 | 서버에서 영상 합성, Supabase Storage 저장 | - |
| 3 | 저장 위치 선택 (하드웨어 / 슬롯) | - |
| 4 | 최종 미리보기 확인 | - |
| 5 | 결제 (슬롯 저장 시) | - |
| 6a | QR 연결 → 프로젝터에 영상 전송 | 프로젝터: QR 표시, Wi-Fi 수신, 영상 재생 |
| 6b | NFC 슬롯에 태깅 | 프로젝터: NFC 슬롯 인식 → 슬롯 번호로 API 호출 → URL 수신 → 재생 |

---

## 10. 보안·권한

- **콘텐츠 접근**: 결제 완료된 사용자만 NFC 슬롯에 기록 가능
- **재생 URL**: Supabase Signed URL(24시간 등) 또는 공개 URL 사용
- **기기 식별**: `device_id`(선택)로 슬롯 이벤트 로깅 가능
